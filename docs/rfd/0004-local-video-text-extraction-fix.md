# RFD 0004: Local Video Text Extraction Fix

## Context

Local testing with `vision/61354d2054ca8878ffe02059f360e7fe.mp4` showed that the MVP did not reliably extract text:

- The video has no embedded subtitle stream.
- The video has an audio stream, but no Whisper CLI is installed.
- The video has visible burned-in captions, but auto mode did not try OCR.
- Full-frame OCR mostly captured platform watermarks and UI text instead of subtitle text.

## Decision

Fix the local video extraction path:

- Auto mode for local media tries `subtitle_track`, then `speech`, then `screen_text`.
- Subtitle extraction probes for subtitle streams before calling FFmpeg.
- OCR uses subtitle-region crops rather than full-frame OCR.
- OCR tries multiple page segmentation modes and filters obvious noise.
- The favicon route returns `204` to remove misleading browser 404 logs.

This is still not a replacement for ASR. If a video contains speech but no readable on-screen text, the server needs Whisper or another ASR provider.

## API or Data Model Impact

No API or database schema changes.

## Failure Modes

- OCR remains best-effort and may misread stylized short-video subtitles.
- Without Whisper, speech-only videos still cannot be accurately transcribed.
- If subtitles are not visible in the sampled crop region, OCR returns fallback text.

## Validation Plan

- Unit test auto plan includes OCR fallback for local media.
- Run unittest and compile checks.
- Run local extraction against the `vision/` video and confirm the transcript is generated by OCR rather than mock fallback.

## Validation Results

- `python3 -m unittest discover -s tests` passed with 6 tests.
- `python3 -m compileall app tests` passed.
- Local workflow test using `vision/61354d2054ca8878ffe02059f360e7fe.mp4` reached `ready_for_review` and generated 3 scripts.
- The local video has no embedded subtitle stream.
- The local workflow returned `screen_text` with provider `ffmpeg-tesseract`, proving that real OCR executed instead of mock fallback.
- OCR quality for the local video remains poor because Tesseract misreads stylized burned-in captions.
- Local whisper.cpp package/model preparation was attempted, but the unpacked Debian binary still fails at runtime because the ggml CPU backend is not registered correctly outside the system install path.
