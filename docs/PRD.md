# 短视频 AI 再创作流水线 PRD

## 1. 背景

用户在刷短视频时，经常会遇到讲得好、数据高、结构清晰的视频。传统做法需要手动记录链接、提取文案、分析结构、重写脚本、配音剪辑和发布，流程长且重复。

本项目希望把这个流程产品化：用户导入一个优质短视频后，系统自动提取文本、分析内容结构、生成原创脚本，并辅助生成数字人口播视频。

## 2. 产品定位

面向内容创作者、矩阵号运营者、知识博主和短视频团队的内容生产工具。

核心价值：

- 快速拆解爆款内容为什么有效。
- 把参考视频转化为原创选题和脚本。
- 降低短视频脚本、配音、字幕和成片制作成本。
- 形成可复用的内容生产流水线。

## 3. 目标用户

### 3.1 个人内容创作者

需要快速产出同领域内容，但缺少稳定选题和脚本能力。

### 3.2 矩阵号运营者

需要批量分析竞品内容，并生成多版本脚本。

### 3.3 企业新媒体团队

需要把行业热点、客户案例或产品卖点转化成短视频内容。

## 4. MVP 目标

第一版验证以下问题：

- 用户是否愿意把参考视频导入系统。
- AI 是否能稳定提取视频核心结构。
- 生成脚本是否能达到可修改、可拍摄的质量。
- 数字人成片是否能减少制作时间。

MVP 不追求一开始全自动发布，优先完成从“导入参考视频”到“生成可审核成片”的闭环。

## 5. 使用流程

### 5.1 标准流程

1. 用户复制短视频链接，粘贴到系统。
2. 系统创建分析任务。
3. 系统获取视频元信息和可访问的视频文件。
4. 系统提取音频。
5. ASR 服务将音频转写成文字。
6. AI 分析脚本结构，包括开头钩子、痛点、观点、案例、转折、结尾引导。
7. AI 生成原创脚本。
8. AI 生成标题、封面文案、标签和发布建议。
9. 用户在审核台修改脚本。
10. 系统调用数字人或 TTS 服务生成视频草稿。
11. 用户确认后导出视频。
12. 用户手动发布，或后续通过官方能力发布。

### 5.2 后续增强流程

- 浏览器插件一键导入当前视频。
- 手机分享菜单导入视频链接。
- 批量导入多个链接。
- 基于账号定位生成不同人设风格。
- 多平台成片规格适配。

## 6. 功能需求

### 6.1 视频导入

支持方式：

- 粘贴视频链接。
- 上传本地视频文件。
- 批量导入链接。

系统需要记录：

- 原视频链接。
- 标题。
- 作者。
- 发布时间。
- 平台。
- 视频时长。
- 用户备注。

说明：收藏夹自动监听不作为 MVP 必选功能。若未来平台提供官方 API 或授权能力，可以作为触发器接入。

### 6.2 音频提取与转写

功能：

- 从视频中提取音频。
- 调用 ASR 服务生成逐字稿。
- 保留时间戳。
- 支持手动修正文稿。

输出：

- 原始转写文本。
- 分段文本。
- 时间戳字幕。

### 6.3 内容分析

AI 需要输出：

- 视频主题。
- 目标受众。
- 内容分类。
- 开头钩子。
- 核心观点。
- 论证结构。
- 情绪节奏。
- 转折点。
- 结尾行动引导。
- 可借鉴点。
- 侵权风险提示。

分析目标是提炼结构和表达方法，不直接复制原文。

### 6.4 原创脚本生成

用户可以选择：

- 内容领域。
- 账号人设。
- 视频时长。
- 表达风格。
- 目标平台。
- 是否口播。
- 是否带货或引流。

系统输出：

- 口播脚本。
- 分镜建议。
- 字幕文案。
- 标题候选。
- 封面文案候选。
- 标签建议。

脚本生成要求：

- 不逐句复刻原文。
- 不保留原作者独特表达。
- 主题可相似，观点和案例应重新组织。
- 支持生成多个版本。

### 6.5 审核台

功能：

- 查看原视频分析结果。
- 查看 ASR 文本。
- 编辑生成脚本。
- 对比不同版本脚本。
- 标记可用、弃用、需重写。
- 触发重新生成。

### 6.6 数字人视频生成

MVP 可以先接第三方数字人平台。

输入：

- 审核后的脚本。
- 数字人形象。
- 配音音色。
- 背景素材。
- 字幕样式。

输出：

- 视频草稿。
- 字幕文件。
- 可下载成片。

### 6.7 发布辅助

MVP 支持：

- 导出视频。
- 复制标题。
- 复制标签。
- 复制简介。
- 记录发布状态。

后续支持：

- 接入平台官方发布 API。
- 定时发布。
- 多账号发布。
- 发布后数据回收。

## 7. 非功能需求

### 7.1 合规

- 不绕过平台访问限制。
- 不未经授权批量抓取内容。
- 不克隆原作者声音和形象。
- 不生成明显抄袭内容。
- 保留人工审核环节。

### 7.2 性能

MVP 目标：

- 3 分钟以内视频，10 分钟内完成分析和脚本生成。
- 数字人成片可异步生成。
- 任务状态可查询。

### 7.3 可扩展性

系统需支持替换：

- ASR 服务。
- 大模型服务。
- 数字人服务。
- 视频生成服务。
- 发布平台。

## 8. 数据对象

- source_type

- platform
- status
- created_at
- updated_at

### 8.2 Transcript

字段：

- id

- segments
- language
- asr_provider
- created_at

### 8.3 Analysis

字段：

- id
- project_id
- topic
- audience
- structure

字段：

## 9. MVP 验收标准

- 用户可以创建项目并导入视频链接或上传视频。
- 系统可以完成音频提取和 ASR 转写。
- 系统可以生成结构化分析结果。
- 系统可以生成至少 3 个原创脚本版本。
- 用户可以在审核台编辑脚本。
- 系统可以调用一个数字人或 TTS 视频服务生成草稿。
- 用户可以下载视频和复制发布文案。

## 10. 暂不做

- 自动监听抖音收藏夹。
- 非官方方式自动发布。
- 自动批量搬运原视频。
- 声音克隆原作者。
- 画面复刻原视频。
- 复杂团队权限系统。
